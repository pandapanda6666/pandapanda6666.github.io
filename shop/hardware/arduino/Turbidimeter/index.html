01 硬體架構與接線

本儀器基於 Arduino Uno 開發，整合了高精度濁度感測器與三鍵互動介面。請嚴格遵守 A0 腳位的電壓範圍，以免燒毀晶片。

控制與顯示腳位

  • 電源按鈕 (PWR)硬體開關 / 模擬器專屬
  • 確定按鈕 (綠)D2 (內部上拉)
  • 取消按鈕 (紅)D3 (內部上拉)
  • 校準按鈕 (黃)D4 (內部上拉)
  • LCD 1602 (I2C)SDA:A4 / SCL:A5

輸出與感測腳位

  • 狀態 LED (綠/黃/紅)D5 / D6 / D7
  • 蜂鳴器 (Passive)D8
  • 濁度計 (Analog)A0 (斷線偵測)
  • 電腦通訊頻率9600 Baud Rate

02 開機與初始化流程

按下電源鍵通電後，系統會播放 1 秒小小世界旋律，LCD 顯示 M00 歡迎畫面。隨後自動進入 M24 資料選擇頁面，詢問使用者如何載入校準參數。

綠色按鈕
讀取 ROM 內部記憶。系統將載入上一次儲存的成功數值，並直接跳轉至測量模式。
黃色按鈕
等待電腦端同步。進入 M25 狀態，等待 Python 軟體透過序列埠發送 SYNC 指令。
紅色按鈕
清除舊有記憶，強制進入四點手動校準流程，從 0 NTU 開始測量。

03 四點校準操作規範

依序對 0, 20, 100, 800 NTU 水樣進行測量。LCD 第二行會即時顯示感測器原始 ADC 值，請等待數值穩定後再操作。

綠色 (確定)

執行 50 次自動採樣並取平均值儲存，成功後自動進入下一個 NTU 測量點。

黃色 (校準)

短按可退回上一個 NTU 點位重測。長按超過 1 秒可直接套用上一次的歷史紀錄值。

紅色 (取消)

短按直接跳過當前點位，該點記錄為 0。後續計算動態模型時將自動忽略無效點。

完成後確認：四點皆完成後，系統會顯示 M26 詢問是否重新校準。按綠色重新來過，按紅色則進入下一步的傳輸詢問。

04 電腦同步與傳輸

本系統支援與 Python 電腦軟體進行雙向標定表格同步。無論何時，電腦端都能即時接收 Arduino 發送的 DATA|Val 封包。

接收電腦參數 (Wait PC Sync)

在 M25 狀態下，Arduino 會每秒發送 REQ_SYNC 請求。當收到 Python 傳來的 SYNC:val1,val2,val3,val4 字串時，系統會自動解析並覆蓋本地表格，隨即進入測量模式。

上傳儀器參數 (Transfer to PC)

在校準完成後的 M27 狀態，若您按下綠色按鈕，Arduino 會向電腦發送 SAVE:val1,val2,val3,val4。Python 軟體收到後會自動更新介面圖表與回歸模型。

05 測量模式與單位

M09 狀態為儀器的常態運作模式。系統會亮起綠燈，並將即時讀取的數值套入動態演算模型，轉換為 NTU 濁度值。

  • 綠色按鈕：短按可切換 LCD 顯示單位，於 NTU 預測值與原始 ADC/電壓值 之間循環切換。
  • 黃色按鈕：中斷測量，重新進入校準流程。
  • 自動回報：每隔 500 毫秒，系統會自動向序列埠輸出格式化的數據流供電腦繪圖。

06 遠端測試模式

為了方便硬體維修與偵錯，系統內建了強大的遠端控制協議。進入方式包含：測量模式下長按任意鍵 3 秒、閒置 5 分鐘自動進入，或由電腦下達「測試模式:開啟」指令。

中文控制指令集

07 錯誤代碼檢索表

當儀器亮起紅燈並發出異常音效時，請對照 LCD 上的 E-Code 進行故障排除。

代碼 顯示文字 異常原因與處置方案
E404 Val Error! 標定邏輯錯誤：當前測量的數值比上一個 NTU 點還要高（水越濁電壓應越低）。請重新採樣，或按紅色按鈕退回上一階。
E405 Btn Missed 按鍵漏按錯誤：在硬體測試階段，未於 5 秒內成功偵測到按鍵訊號。請檢查線路是否斷開（持續維持高電位）。
E406 Btn Stuck 按鍵卡死錯誤：系統發現某顆按鈕被持續按壓超過 1 分鐘。請檢查實體按鈕是否卡住，或電路是否發生對地短路。
E503 Snsr Error! 感測器異常斷線：A0 腳位讀值大於 1020。由於系統啟用了 INPUT_PULLUP，代表感測器線路已拔除或斷裂。請重新插緊訊號線。
